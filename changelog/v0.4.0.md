# v0.4.0 Changelog Draft

Status: **draft** — content is finalised against the v0.4 release cut.

## Theme

**Capsule Bus Experimental** — governed event-to-command routing inside a single Opstage CE instance, default-disabled, gated by `OPSTAGE_CAPSULE_BUS_ENABLED=true`.

## Added

- `xtrape-capsule-contracts-node`: `BusRoutingRule`, `BusEventEnvelope` Zod schemas. New `ErrorCode` values: `BUS_DISABLED`, `BUS_DEPTH_EXCEEDED`, `BUS_RATE_LIMITED`, `BUS_ROUTE_NOT_FOUND`.
- `xtrape-capsule-ce` backend: feature-flagged bus subsystem.
  - Migrations: `bus_routing_rules`, `bus_events` tables; `commands.sourceEventId` FK column.
  - Admin endpoints: `GET / POST / PATCH / DELETE /api/admin/bus/routes(/:id)`, `GET /api/admin/bus/events`, `GET /api/admin/bus/audit`.
  - Agent-authenticated endpoint: `POST /api/agents/{agentId}/bus/events` (server-overwrites `sourceServiceCode` from agent identity).
  - In-process router with hard `max routing depth = 1` and per-agent ingestion rate limit (default `60/min`).
  - Audit events: `bus.event.received`, `bus.route.matched`, `bus.command.created`, `bus.route.failed`, `bus.route.unmatched`.
- `xtrape-capsule-ce` UI: `/bus/routes` page (CRUD + status toggle); `/bus/events` page (recent events + linked audit).
- `xtrape-capsule-agent-node`: `CapsuleAgent.publishEvent(envelope)` method. Typed errors `BusDisabledError`, `BusRateLimitedError`.
- `xtrape-capsule-demo`: `notify` action; `createDemoItem` now publishes `demo.item.created` events; `docs/v0.4-bus-smoke-test.md` runbook.
- `xtrape-capsule-site`: `concepts/capsule-bus.md`, routing-rule reference, first-bus-rule tutorial, v0.4 release notes.

## Configuration

| Env var | Default | Purpose |
| --- | --- | --- |
| `OPSTAGE_CAPSULE_BUS_ENABLED` | `false` | Master feature flag. When off, admin endpoints return 503, agent endpoint returns 404, UI page hidden. |
| `OPSTAGE_CAPSULE_BUS_INGEST_PER_MIN` | `60` | Per-agent rate limit for event ingestion. |

## Breaking changes

None at the wire level. v0.3 agents and CE work unchanged against a v0.4 CE backend — the bus is purely additive.

The Agent SDK's `publishEvent(...)` is new; nothing existing is removed or renamed.

## Migration from v0.3

The schema migration is automatic on first start. Operators upgrading from `0.3.x`:

1. Pull `ghcr.io/xtrape-com/xtrape-capsule-ce:0.4.0`.
2. Start once with `OPSTAGE_CAPSULE_BUS_ENABLED` **unset** (default `false`) to confirm baseline still passes.
3. Set `OPSTAGE_CAPSULE_BUS_ENABLED=true` and restart.
4. Create a first route in `DRY_RUN` status; verify audits.
5. Promote to `ENABLED`.

## Non-goals (deferred to v0.5+)

- Fan-out (commands-per-event > 1).
- Capsule Catalog: Service Manifest v1, endpoint visibility, capability metadata, discovery-by-capability — all in **v0.5**.
- Public Capsule Registry across Opstage instances — **v0.6**.
- Private Capsule Marketplace concepts — **v0.7**.
- External message brokers (Redis / Kafka / NATS / RabbitMQ).
- Service mesh, mTLS-everywhere, load balancing.
- Non-Node OpHub runtime (Go etc.) — pushed to v0.5+ when routing contract is proven stable enough to justify a second-runtime implementation.

## Known limitations

- `xtrape-capsule-ce` Docker image is `linux/amd64` only (carried over from v0.2 — `arm64` re-introduction tracked for v0.5).
- Bus events are not replayable; CE retains them for diagnostics only.
- Route count > 100 produces a UI warning; eval cost is unbounded by design for v0.4.

## Audit-event reference

| Action | Emitted when |
| --- | --- |
| `bus.event.received` | Always, on every successful ingestion. |
| `bus.route.matched` | Per matched enabled route (any status). |
| `bus.command.created` | A command was created from a matched, `ENABLED` route. Preceded by `bus.route.matched`. |
| `bus.route.failed` | Route matched but command creation failed. Preceded by `bus.route.matched`. |
| `bus.route.unmatched` | Event ingested but no enabled route matched. |

See `04-opstage/v0.4-capsule-bus-experimental.md` for the canonical pairing rules.
