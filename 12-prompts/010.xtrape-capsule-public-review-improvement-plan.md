# xtrape-capsule Public Review Improvement Plan

Date: 2026-05-06

## Reviewed Repositories

```text
xtrape-capsule-ce
xtrape-capsule-site
xtrape-capsule-agent-node
xtrape-capsule-contracts-node
xtrape-capsule-docs
```

## Goal

Prepare the public-facing `xtrape-capsule` GitHub project matrix for a future:

```text
v0.1.0 Public Preview
```

The target public positioning is:

> Xtrape Capsule is a lightweight, self-hosted, agent-based runtime governance control plane for Capsule Services.

Expanded public wording:

> Xtrape Capsule helps teams connect small services, automation workers, integration adapters, background jobs, private tools, and agent runtimes to a unified Opstage console through an embedded Agent SDK.

Avoid positioning Xtrape Capsule as:

```text
a CAPI platform
a CAPI control plane
a platform for unofficial APIs
a generic config center
a generic monitoring dashboard
a Backstage replacement
```

### Why

CAPI is not part of the public Xtrape Capsule positioning. It should not appear in public README, docs, homepage, navigation, examples, or use cases.

### Affected Public Repositories

```text
xtrape-capsule-site
xtrape-capsule-ce
xtrape-capsule-agent-node
xtrape-capsule-contracts-node
```

### Required Search

Run in every public repository:

```bash
rg -n "CAPI|capi|capi-service|capi-services|capi bridge|CAPI bridge|capi-chatgpt"
```

## P0-3. Fix `xtrape-capsule-site` Quick Start

### Problem

The homepage currently shows a GHCR `latest` command, but the public image/package status is unclear. If the image is not published, this will break the first user experience.

### Required Decision

Choose one mode before Public Preview.

## Mode A - Source Build Only

Use:

```bash
git clone https://github.com/xtrape-com/xtrape-capsule-ce.git
cd xtrape-capsule-ce
docker compose -f deploy/compose/docker-compose.yml up --build -d
```

State clearly:

```text
Public Docker images are planned for v0.1.0 Public Preview.
```

## Mode B - Published GHCR Image

Use a fixed version tag:

```bash
docker run -p 8080:8080 -v opstage-data:/app/data ghcr.io/xtrape-com/xtrape-capsule-ce:0.1.0
```

Avoid using `latest` as the only documented command.

### Acceptance Criteria

```text
[ ] Homepage Quick Start and detailed Quick Start use the same deployment path.
[ ] No command references an unavailable image.
[ ] Quick Start works in a clean environment.
[ ] CE README uses the same deployment path as the site.
```
