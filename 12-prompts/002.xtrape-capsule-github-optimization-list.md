# xtrape-capsule GitHub 项目矩阵优化清单

> 目标：将 `xtrape-capsule` 项目矩阵完善到适合 `v0.1.0 Public Preview` 对外推广的状态。  
> 适用仓库：
>
> - `xtrape-capsule-ce`
> - `xtrape-capsule-site`

---

## 1. 当前总体判断

当前项目已经具备初始产品形态，但还没有完全达到公开推广标准。

目前状态：

| 仓库 | 当前状态 | 推广准备度 |
|---|---|---:|
| `xtrape-capsule-site` | 已具备 public docs/site 基础形态 | 75% |
| `xtrape-capsule-ce` | 功能完整度较好，但 README 偏工程说明 | 55% |
| `xtrape-capsule-agent-node` | 核心 SDK 仓库，但 README 太短 | 35% |
| `xtrape-capsule-contracts-node` | 核心协议仓库，但 README 太短 | 35% |

当前适合：

```text
小范围 review / 熟人试用 / AI 继续优化
```

当前不建议：

```text
大范围公开推广 / Hacker News / Product Hunt / 大规模社媒宣传
```

示例形式参考：

```ts
import { CapsuleAgent } from "@xtrape/capsule-agent-node";

const agent = new CapsuleAgent({
  backendUrl: process.env.OPSTAGE_BACKEND_URL!,
  registrationToken: process.env.OPSTAGE_REGISTRATION_TOKEN,
  tokenFile: "./data/agent-token.json",
});

agent.registerService({
  code: "hello-capsule",
  name: "Hello Capsule",
  version: "0.1.0",
  runtime: "nodejs",
});

agent.registerHealthProvider(async () => ({
  status: "UP",
  details: {
    uptime: process.uptime(),
  },
}));

agent.registerAction({
  name: "echo",
  label: "Echo",
  prepare: async () => ({
    schema: {
      type: "object",
      properties: {
        message: { type: "string" },
      },
    },
  }),
  execute: async (payload) => ({
    result: {
      echo: payload.message,
    },
  }),
});

await agent.start();
```

> 注意：上面只是结构示例，最终必须根据仓库真实 SDK API 调整。
