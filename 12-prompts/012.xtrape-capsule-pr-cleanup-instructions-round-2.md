## P0-2. Align HealthStatus and EffectiveStatus terminology

### Problem

Docs currently mix these two vocabularies without clearly distinguishing them:

```text
UP / DEGRADED / DOWN / UNKNOWN
HEALTHY / UNHEALTHY / STALE / OFFLINE
```

### Required model

Use this distinction everywhere:

```text
HealthStatus
  Protocol-level health reported by Capsule Services and Agents.
  Values: UP / DEGRADED / DOWN / UNKNOWN

EffectiveStatus
  Operator-facing status derived by Opstage from health reports, heartbeat, and freshness.
  Values: HEALTHY / UNHEALTHY / STALE / OFFLINE
```

### Required wording

Add this explanation wherever both concepts appear:

```md
Agent health providers return protocol-level `HealthStatus` values:
`UP`, `DEGRADED`, `DOWN`, `UNKNOWN`.

Opstage may derive an operator-facing `effectiveStatus`:
`HEALTHY`, `UNHEALTHY`, `STALE`, `OFFLINE`.
```

### Acceptance criteria

```text
[ ] CE README does not imply HEALTHY/UNHEALTHY/STALE/OFFLINE are protocol health values.
[ ] Agent SDK README explains protocol-level HealthStatus.
[ ] Contracts README owns and documents protocol-level HealthStatus.
[ ] Site docs explain that effectiveStatus is derived by Opstage.
```

### Problem

This is invalid or at least unsafe for VitePress parsing.

```md

layout: home

  name: Xtrape Capsule
  text: A lightweight control plane for AI-era Capsule Services.
  tagline: Connect small services, automation workers, integration services, and agent runtimes to a unified Opstage console through an embedded Agent SDK.
  actions:
    - theme: brand
      text: Get Started
      link: /getting-started/quick-start
    - theme: alt
      text: GitHub
      link: https://github.com/xtrape-com/xtrape-capsule-ce
features:
  - title: Agent-based governance
    details: Capsule Services connect to Opstage through an embedded Agent SDK.
  - title: Self-hosted CE
    details: Run the CE stack locally with SQLite and Docker Compose.
  - title: Runtime visibility
    details: Track agents, services, health, config visibility, actions, and audit events.
---

::: warning Public Review
Xtrape Capsule is currently in **Public Review** before the `v0.1.0 Public Preview` release.
:::
```

Adjust feature text as needed, but keep valid frontmatter.

### Acceptance criteria
