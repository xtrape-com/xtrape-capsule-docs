## P0-3. Align HealthStatus and EffectiveStatus terminology

### Problem

Docs currently mix two status vocabularies:

```text
UP / DEGRADED / DOWN / UNKNOWN
HEALTHY / UNHEALTHY / STALE / OFFLINE
```

This is confusing unless the distinction is made explicit.

### Required Model

Use this distinction everywhere:

```text
HealthStatus
  Protocol-level health reported by Capsule Services / Agents.
  Values: UP / DEGRADED / DOWN / UNKNOWN

EffectiveStatus
  Operator-facing status derived by Opstage from health reports, heartbeat, and freshness.
  Values: HEALTHY / UNHEALTHY / STALE / OFFLINE
```

### Required Action

Add this explanation wherever both concepts appear:

```md
Agent health providers return protocol-level `HealthStatus` values:
`UP`, `DEGRADED`, `DOWN`, `UNKNOWN`.

Opstage may derive an operator-facing `effectiveStatus`:
`HEALTHY`, `UNHEALTHY`, `STALE`, `OFFLINE`.
```

### Acceptance Criteria

```text
[ ] CE README no longer implies HEALTHY/UNHEALTHY/STALE/OFFLINE are the protocol health values.
[ ] Agent SDK README explains protocol HealthStatus.
[ ] Contracts README explains HealthStatus clearly.
[ ] Site docs explain how Opstage derives effective status.
```
