# Remove Public CAPI References from xtrape-capsule Repositories

## Purpose

The current public-facing `xtrape-capsule` GitHub repositories should not mention **CAPI**.

CAPI is an internal or future business/application scenario. It is **not part of the public definition of
xtrape-capsule**. Public references to CAPI may make users misunderstand `xtrape-capsule` as a CAPI-specific platform
instead of a general-purpose Capsule Service runtime governance system.

The public positioning should remain:

> Xtrape Capsule is a lightweight, self-hosted, agent-based runtime governance control plane for Capsule Services.

---

## Scope

Apply this cleanup to the following public repositories:

```text
xtrape-capsule-site
xtrape-capsule-ce
xtrape-capsule-agent-node
xtrape-capsule-contracts-node
```

Primary focus:

```text
README.md
docs/**/*.md
docs/.vitepress/config.ts
blog/**/*.md
examples/**/*.md
package metadata if applicable
navigation/sidebar labels
use case pages
```

---

## Global Rule

Do not mention the following terms in public README, docs, website pages, examples, navigation, sidebar, or use case pages:

```text
CAPI
capi
CAPI services
capi-services
CAPI bridges
capi bridge
CAPI use case
capi service governance
```

Public repositories should use general-purpose terms instead.

---

## Recommended Replacement Terms

| Remove / Replace | Recommended Replacement |
|---|---|
| CAPI services | integration services |
| CAPI bridges | integration adapters |
| CAPI use case | integration services use case |
| CAPI service governance | lightweight service governance |
| CAPI / Playwright / account pool | integration services / Playwright workers / account pool |
| AI-era CAPI services | AI-era integration and automation services |
| capi-services.md | integration-services.md |

Preferred public vocabulary:

```text
small services
automation workers
integration services
integration adapters
background jobs

agent runtimes
stateful resource services

```

```text
Xtrape Capsule is a lightweight, self-hosted, agent-based runtime governance control plane for Capsule Services.
```

```text
Xtrape Capsule helps teams connect small services, automation workers, integration adapters, background jobs, private tools, and agent runtimes to a unified Opstage console through an embedded Agent SDK.
```
