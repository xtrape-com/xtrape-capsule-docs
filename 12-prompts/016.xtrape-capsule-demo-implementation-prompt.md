## Project Purpose

`xtrape-capsule-demo` is a public demo repository for Xtrape Capsule.

Its purpose is to provide a complete, runnable example Capsule Service that connects to Xtrape Capsule Opstage CE through the Node.js Embedded Agent SDK.

This repository is not the Opstage platform itself. It is not an SDK package. It is a sample workload / reference Capsule Service.

One-sentence positioning:

```text
Demo Capsule Services for testing and learning Xtrape Capsule Opstage integration.
```

## Actions

Implement actions in `src/actions.ts`.

### Action 1: `echo`

Purpose:

```text

```

Prepare result should define input schema:

```json
{
  "type": "object",
  "required": ["message"],
  "properties": {
    "message": {
      "type": "string",
      "title": "Message"
    }
  }
}
```

Execute input:

```json
{
  "message": "hello"
}
```

Execute result:

```json
{
  "echo": "hello",
  "service": "demo-worker"
}
```

---

### Action 2: `runDemoJob`

Purpose:

```text
Simulate a short background job and increment jobCounter.
```

Prepare result should define:

```json
{
  "type": "object",
  "properties": {
    "durationMs": {
      "type": "number",
      "title": "Duration in milliseconds",
      "default": 1000
    }
  }
}
```

Execute behavior:

```text
1. Wait durationMs, clamped between 100 and 5000.
2. Increment jobCounter.
3. Return jobCounter and execution duration.
```

Execute result:

```json
{
  "jobCounter": 1,
  "durationMs": 1000,
  "status": "completed"
}
```

---

### Action 3: `setHealthMode`

Purpose:

```text
Change the demo health mode.
```

Prepare result should define:

```json
{
  "type": "object",
  "required": ["mode"],
  "properties": {
    "mode": {
      "type": "string",
      "enum": ["healthy", "degraded", "unhealthy"]
    }
  }
}
```

Execute behavior:

```text
Set state.healthMode to the requested mode.
```

Execute result:

```json
{
  "healthMode": "degraded"
}
```

---

### Action 4: `failOnce`

Purpose:

```text
Demonstrate action failure and audit visibility.
```

Prepare result may have no required input.

Execute behavior:

```text
Throw a controlled error with message:
"Demo failure triggered by failOnce action"
```

This allows users to see failed command/action results in Opstage.

## Agent Integration

Implement agent setup in `src/agent.ts`.

Use environment variables:

```env
OPSTAGE_BACKEND_URL=http://localhost:8080
OPSTAGE_REGISTRATION_TOKEN=
OPSTAGE_AGENT_TOKEN_FILE=./data/agent-token.json
DEMO_SERVICE_CODE=demo-worker
DEMO_SERVICE_NAME=Demo Worker Service
DEMO_SERVICE_VERSION=0.1.0
DEMO_HTTP_PORT=3101
```

Required behavior:

```text
1. Load environment variables.
2. Initialize Capsule Agent.
3. If registration token is provided and no stored agent token exists, register with Opstage.
4. Store agent token in OPSTAGE_AGENT_TOKEN_FILE.
5. Report demo-worker service manifest.
6. Register health reporter.
7. Register config reporter.
8. Register action catalog/prepare/execute handlers.
9. Start command polling or SDK-managed loop.
10. Log useful startup messages.
```

Startup logs should include:

```text
Demo Worker Service starting...
Opstage backend URL: ...
Service code: demo-worker
Agent token file: ...
Waiting for registration / heartbeat...
```

Do not print secrets.

Mask registration token if logging is necessary:

```text
abcd...wxyz
```
