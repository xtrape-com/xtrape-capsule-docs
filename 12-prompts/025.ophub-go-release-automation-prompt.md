# Prompt: Prepare GitHub Automatic Release for `xtrape-capsule-ophub-go`

你将作为代码实现助手，帮助完善 `xtrape-com/xtrape-capsule-ophub-go` 的 v0.3.0 自动发布准备工作。

目标：为 Go 版本的 OpHub runtime 增加稳定、可重复、适合 public review / v0.3 release train 的 GitHub 自动 release 流程。

---

## 0. 背景

`xtrape-capsule-ophub-go` 是 xtrape-capsule 体系中的 OpHub Go runtime。

v0.3 release train 的推荐顺序是：

```text
1. contracts-node
2. agent-node
3. xtrape-capsule-ce
4. xtrape-capsule-ophub-go
5. xtrape-capsule-demo
6. xtrape-capsule-site
```

OpHub Go 在 CE 与 demo 之间发布，因为 demo 的 OpHub-managed adapter mode 需要依赖 OpHub runtime 来验证完整链路。

当前目标不是引入复杂产品能力，而是补齐发布工程能力：

- 自动构建跨平台二进制；
- 自动生成 GitHub Release；
- 上传 release assets；
- 生成 checksums；
- 可选发布 Docker image 到 GHCR；
- README / docs 中说明安装和验证方式。

## 4. `.goreleaser.yaml` 要求

新增 `.goreleaser.yaml`。

基础要求：

- 构建 Linux / macOS / Windows；
- 构建 amd64 / arm64；
- Windows 产物带 `.exe`；
- 生成 checksums；
- 生成 GitHub Release；
- release name 使用 tag；
- snapshot 支持本地验证；
- 构建产物命名清晰；
- 不要发布未确认的 npm / Node 产物；
- 不要引入和 Go runtime 无关的内容。

建议目标产物：

```text
ophub_Darwin_arm64.tar.gz
ophub_Darwin_x86_64.tar.gz
ophub_Linux_arm64.tar.gz
ophub_Linux_x86_64.tar.gz
ophub_Windows_x86_64.zip
checksums.txt
```

注意：实际命名以 GoReleaser 输出为准，但应保持可读。

参考配置方向：

```yaml
version: 2

project_name: xtrape-capsule-ophub-go

before:
  hooks:
    - go mod tidy
    - go test ./...

builds:
  - id: ophub
    binary: ophub
    main: ./cmd/ophub
    env:
      - CGO_ENABLED=0
    goos:
      - linux
      - darwin
      - windows
    goarch:
      - amd64
      - arm64
    ignore:
      - goos: windows
        goarch: arm64
    ldflags:
      - -s -w
      - -X main.version={{.Version}}
      - -X main.commit={{.Commit}}
      - -X main.date={{.Date}}

archives:
  - id: ophub
    builds:
      - ophub
    format_overrides:
      - goos: windows
        format: zip
    name_template: >-
      ophub_
      {{- title .Os }}_
      {{- if eq .Arch "amd64" }}x86_64
      {{- else }}{{ .Arch }}{{ end }}

checksum:
  name_template: "checksums.txt"

snapshot:
  version_template: "{{ incpatch .Version }}-next"

changelog:
  sort: asc
  filters:
    exclude:
      - "^docs:"
      - "^test:"
      - "^chore:"
```

重要：`main: ./cmd/ophub` 只是示例。必须根据仓库实际 entrypoint 修正。

如果项目目前没有 version 注入变量，不要强行加入无法编译的 `-X main.version`。可以先检查代码中是否存在：

```go
var version string
var commit string
var date string
```

如果不存在，有两种选择：

1. 去掉对应 ldflags；
2. 增加一个小的 version 包，例如 `internal/version`，并让 CLI 输出版本。

推荐实现一个最小版本能力：

```bash
ophub --version
```

但不要因此扩大 scope。

## 5. GitHub Release Workflow

新增或更新：

```text
.github/workflows/release.yml
```

要求：

- 仅在 tag push 时触发；
- tag 格式：`v*`;
- 使用 GoReleaser official action；
- 使用 `GITHUB_TOKEN`;
- 权限包含 `contents: write`;
- 使用 Node 无关流程；
- 使用当前稳定 Go 版本，例如 `1.22` 或仓库 `go.mod` 指定版本。

示例：

```yaml
name: Release

on:
  push:
    tags:
      - "v*"

permissions:
  contents: write
  packages: write

jobs:
  goreleaser:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Set up Go
        uses: actions/setup-go@v5
        with:
          go-version-file: go.mod
          cache: true

      - name: Run GoReleaser
        uses: goreleaser/goreleaser-action@v6
        with:
          distribution: goreleaser
          version: latest
          args: release --clean
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

如果不做 Docker image，可移除 `packages: write`。

## 6. CI Workflow

如果仓库没有 CI，请新增：

```text
.github/workflows/ci.yml
```

要求：

- PR 和 main push 触发；
- 执行 `go test ./...`;
- 执行 `go vet ./...`；
- 执行 `go build`；
- 执行 GoReleaser snapshot 检查。

示例：

```yaml
name: CI

on:
  pull_request:
  push:
    branches:
      - main

permissions:
  contents: read

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Go
        uses: actions/setup-go@v5
        with:
          go-version-file: go.mod
          cache: true

      - name: Test
        run: go test ./...

      - name: Vet
        run: go vet ./...

      - name: Build
        run: go build ./...

      - name: GoReleaser snapshot
        uses: goreleaser/goreleaser-action@v6
        with:
          distribution: goreleaser
          version: latest
          args: release --snapshot --clean --skip=publish
```

注意：如果 GoReleaser v2 对 `--skip=publish` 的参数有兼容问题，请使用当前官方推荐参数，并确保 CI 通过。
