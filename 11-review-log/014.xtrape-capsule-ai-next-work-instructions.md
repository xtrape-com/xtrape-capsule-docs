### Known issue

Codex review identified that `pnpm/action-setup@v4` may need an explicit version when the workflow uses sibling checkout layout and no root `package.json` is available at setup time.

### Required fix

Set the pnpm version explicitly:

```yaml
- name: Setup pnpm
  uses: pnpm/action-setup@v4
  with:
    version: 9
```

Use the actual pnpm major version used by the project if different.

### Required dependency handling

Agent SDK depends on unpublished:

```text
@xtrape/capsule-contracts-node
```

The CI must handle this deterministically.

Acceptable approaches:

```text
Option A:
  Checkout contracts-node as a sibling repository and use pnpm workspace / local link.

Option B:
  Build contracts-node first, pack it with pnpm pack, then install the tarball in agent-node.

Option C:
  Publish contracts-node first, then use npm dependency.
```

For current Public Review, Option A or B is acceptable.


### Known issue to verify

The current README says the SDK stores issued credentials as:

```text
<agentId>:<agentToken>
```

But the PR description mentions:

```text
JSON credential persistence
legacy token-only persistence
```

This may indicate inconsistency between README, implementation, and tests.

### Required decision

Choose one canonical token file format.

## Option A - Colon format

Use if implementation stores:

```text
<agentId>:<agentToken>
```

Then ensure:

```text
[ ] README says `<agentId>:<agentToken>`.
[ ] FileTokenStore tests assert `<agentId>:<agentToken>`.
[ ] No JSON token format is described as primary.
[ ] Any legacy behavior is clearly documented if it exists.
```

## Option B - JSON format

Use if implementation stores JSON:

```json
{
  "agentId": "agt_...",
  "agentToken": "..."
}
```

Then ensure:

```text
[ ] README is updated to JSON format.
[ ] FileTokenStore tests assert JSON.
[ ] Migration/legacy behavior is documented if supported.
```

Do not leave README and tests inconsistent.

### Required tests

```text
[ ] Config provider reporting.
[ ] Action prepare.

[ ] Unknown action handler behavior.
[ ] Command polling / command result reporting.
[ ] Token values are not logged or exposed in normal output.
```
