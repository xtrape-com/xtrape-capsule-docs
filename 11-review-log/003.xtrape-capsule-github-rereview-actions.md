# P0 - Must Fix Before Public Preview

## P0-1. Remove All Public CAPI References

### Why

CAPI is not part of the public `xtrape-capsule` positioning. It should not appear in public README, docs, homepage, navigation, sidebar, examples, or use cases.

Public CAPI references make users think Xtrape Capsule is a CAPI-specific platform, which weakens its general-purpose Capsule Service positioning.

### Found Issues

The following public files still contain CAPI references:

```text
xtrape-capsule-site/docs/index.md
xtrape-capsule-site/docs/.vitepress/config.ts
xtrape-capsule-site/docs/faq.md
xtrape-capsule-site/README.md
xtrape-capsule-ce/README.md
```

Examples to remove:

```text
CAPI services
CAPI bridges
/use-cases/capi-services
CAPI Services
capi-chatgpt
```

### Required Replacements

| Current | Replace With |
|---|---|
| CAPI services | integration services |
| CAPI bridges | integration adapters |
| /use-cases/capi-services | /use-cases/integration-services |
| CAPI Services | Integration Services |
| capi-chatgpt | demo-worker or integration-worker |

### Required Actions

#### In `xtrape-capsule-site/docs/index.md`

Replace:

```text
Connect small services, automation workers, CAPI services, and agent runtimes...
```

with:

```text
Connect small services, automation workers, integration services, and agent runtimes to a unified Opstage console through an embedded Agent SDK.
```

Replace:

```text
CAPI bridges, Playwright workers, account pools, OTP readers, proxy checkers, and AI Agent runtimes
```

with:

```text
integration adapters, Playwright workers, account pools, background jobs, private tools, and AI Agent runtimes
```

with:

### Required Minimal Example

Provide a real, tested example matching the current SDK API:

```ts
import { CapsuleAgent } from "@xtrape/capsule-agent-node";

const agent = new CapsuleAgent({
  backendUrl: process.env.OPSTAGE_BACKEND_URL ?? "http://localhost:8080",
  registrationToken: process.env.OPSTAGE_REGISTRATION_TOKEN,
  tokenFile: process.env.OPSTAGE_AGENT_TOKEN_FILE ?? "./data/agent-token.json",
});

agent.registerService({
  code: "my-capsule",
  name: "My Capsule Service",
  version: "0.1.0",
});

agent.registerHealthCheck("my-capsule", async () => ({
  status: "HEALTHY",
  message: "ok",
}));

agent.registerAction("my-capsule", {
  name: "echo",
  label: "Echo",
  inputSchema: {
    type: "object",
    required: ["message"],
    properties: {
      message: { type: "string" },
    },
  },
  handler: async (payload) => ({
    echoed: payload.message,
  }),
});

await agent.start();
```

Adjust this sample if the actual SDK API differs.
