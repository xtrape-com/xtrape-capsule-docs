# xtrape-capsule npm Dependency Migration AI Operation Guide

Date: 2026-05-06

## Purpose

Move `xtrape-capsule` from an inconsistent multi-repository `workspace:*` dependency setup to a proper npm-based dependency chain.

Current repositories:

```text
xtrape-capsule-contracts-node
xtrape-capsule-agent-node
xtrape-capsule-ce
```

are separate GitHub repositories, but `agent-node` and `CE` currently use `workspace:*` dependencies that only make sense inside a monorepo.

This causes or may cause:

```text
1. fresh clone install failures;
2. pnpm workspace package not found errors;
3. confusing CE workspace declarations;
4. CI hacks using sibling checkout or local mirrors;
5. README / actual directory structure mismatch;
6. poor Public Review experience.
```

The correct public/open-source dependency chain should be:

```text
@xtrape/capsule-contracts-node
        ↓
@xtrape/capsule-agent-node
        ↓
xtrape-capsule-ce
```

---

# Target Strategy

Use npm packages.

Recommended prerelease version:

```text
0.1.0-public-review.0
```

Recommended npm dist-tag:

```text
public-review
```

Later, after Public Review feedback is processed, publish:

```text
0.1.0
```

with the normal `latest` tag.

```text
1. Do not change product positioning.
2. Do not add new features.
3. Do not rewrite README except where dependency/install status is wrong.
4. Do not use workspace:* for packages that are not physically present in the same repository.
5. Do not keep pnpm-workspace.yaml entries pointing to nonexistent directories.
6. Every repo must support fresh clone + pnpm install.
7. Each PR must have one clear purpose.
```

# Phase 1 - Prepare and Publish Contracts Package

Repository:

```text
xtrape-capsule-contracts-node
```

Package:

```text
@xtrape/capsule-contracts-node
```

## 1.1 Check package metadata

Verify `package.json` contains:

```json
{
  "name": "@xtrape/capsule-contracts-node",
  "version": "0.1.0-public-review.0",
  "description": "TypeScript contracts and Zod schemas for Xtrape Capsule and Opstage wire protocols.",
  "license": "Apache-2.0",
  "main": "./dist/index.cjs",
  "module": "./dist/index.js",
  "types": "./dist/index.d.ts",
  "files": [
    "dist",
    "spec",
    "README.md",
    "LICENSE",
    "NOTICE"
  ],
  "publishConfig": {
    "registry": "https://registry.npmjs.org/",
    "access": "public"
  }
}
```

Adjust based on the existing project convention if needed.

Also verify:

```text
repository
homepage
bugs
keywords
```

are present and point to the correct GitHub repository and public docs.

## 1.2 Verify exports

Confirm package exports work correctly.

Recommended `package.json` pattern:

```json
{
  "exports": {
    ".": {
      "types": "./dist/index.d.ts",
      "import": "./dist/index.js",
      "require": "./dist/index.cjs"
    }
  }
}
```

Only add or modify exports if the package already builds both ESM and CJS outputs.

Do not invent paths that are not emitted by the build.

## 1.3 Run validation

Run:

```bash
pnpm install
pnpm build
pnpm typecheck
pnpm test
pnpm pack --dry-run
```

If `pnpm pack --dry-run` is not available or not supported in the environment, run:

```bash
pnpm pack
tar -tf xtrape-capsule-contracts-node-*.tgz
```

and inspect contents.

## 1.4 Inspect package contents

Package should include:

```text
dist/
spec/
README.md
LICENSE
NOTICE
```

Package should not include:

```text
src/
tests/
node_modules/
.github/
coverage/
.env
.DS_Store
```

unless there is a deliberate reason.

## 1.5 Publish to npm

Before publishing, confirm npm login:

```bash
npm whoami
```

Publish prerelease:

```bash
npm publish --access public --tag public-review
```

## 1.6 Verify npm publication

Run:

```bash
npm view @xtrape/capsule-contracts-node@public-review version
npm view @xtrape/capsule-contracts-node@0.1.0-public-review.0
```

Expected:

```text
0.1.0-public-review.0
```

If publication fails due to missing npm organization/scope permissions, stop and report the exact error.

Do not switch to an alternative package name without confirmation.

# Phase 3 - Update CE to Use npm Dependencies

Repository:

```text
xtrape-capsule-ce
```

Goal:

```text
CE should no longer use workspace:* for packages that are not physically part of CE.
```

## 3.1 Replace package dependencies

Search all package.json files:

```bash
rg -n '"@xtrape/capsule-(agent-node|contracts-node)"'
```

Likely files:

```text
package.json
apps/opstage-backend/package.json
apps/demo-capsule-service/package.json
```

Replace:

```json
"@xtrape/capsule-agent-node": "workspace:*"
"@xtrape/capsule-contracts-node": "workspace:*"
```

with:

```json
"@xtrape/capsule-agent-node": "^0.1.0-public-review.0"
"@xtrape/capsule-contracts-node": "^0.1.0-public-review.0"
```

Use the actual published prerelease version.

## 3.2 Fix pnpm-workspace.yaml

Current incorrect form may look like:

```yaml
packages:
  - 'xtrape-capsule-contracts-node'
  - 'xtrape-capsule-agent-node'
  - 'packages/*'
```

Replace with paths that actually exist in CE.

Recommended:

```yaml
packages:
  - 'apps/*'
  - 'packages/*'
```

If CE uses additional real workspace paths, include them.

Do not list nonexistent paths.

## 3.3 Fix README Development structure

If README says:

```text
packages/contracts         # workspace mirror of @xtrape/capsule-contracts-node
packages/agent-node        # workspace mirror of @xtrape/capsule-agent-node
```

remove or replace it.

Suggested replacement:

```text
packages/db                # Prisma schema and database package
packages/shared            # shared UI/domain types used by CE
packages/test-utils        # test utilities
```

Add a note if helpful:

```md
`@xtrape/capsule-contracts-node` and `@xtrape/capsule-agent-node` are consumed as npm packages during Public Review.
```

## 3.4 Update contracts check

Open:

```text
scripts/external-contracts-check.mjs
```

Current logic may enforce or expect `workspace:*`.

Change it to validate the npm dependency strategy.

Rules should be:

```text
1. CE package.json must not depend on @xtrape/capsule-agent-node via workspace:* unless a matching local workspace package exists.
2. Backend package.json must not depend on @xtrape/capsule-contracts-node via workspace:* unless a matching local workspace package exists.
3. Demo capsule service must not depend on @xtrape/capsule-agent-node via workspace:* unless a matching local workspace package exists.
4. Dependencies should use ^0.1.0-public-review.x or ^0.1.0 during Public Review.
5. pnpm-workspace.yaml should only list real local directories.
```

Suggested allowed pattern:

```text
^0.1.0-public-review.0
^0.1.0-public-review.1
^0.1.0
```

Implement robust semver-ish checks. Do not hardcode only one prerelease version if avoidable.

## 3.5 Update lockfile

Run:

```bash
pnpm install
```

Commit updated:

```text
pnpm-lock.yaml
```

## 3.6 Validate CE

Run:

```bash
pnpm install
pnpm contracts:check
pnpm db:validate
pnpm typecheck
pnpm build
pnpm test
pnpm release:check
```

If `pnpm test` is not stable yet, report exact reason.

Do not hide failures.

## 3.7 Validate fresh clone behavior

Use a clean directory if possible:

```bash
cd /tmp
git clone https://github.com/xtrape-com/xtrape-capsule-ce.git
cd xtrape-capsule-ce
pnpm install
pnpm contracts:check
pnpm db:validate
pnpm typecheck
pnpm build
```

If testing a local branch before merge, clone and checkout the branch.

# Phase 4 - Update CI Workflows if Needed

## 4.1 Agent CI

If agent CI currently uses sibling checkout of contracts-node only because of unpublished package dependency, simplify it after npm publication.

Preferred CI after npm publication:

```yaml
name: CI

on:
  pull_request:
  push:
    branches: [main]

permissions:
  contents: read

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 9

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm

      - run: pnpm install --frozen-lockfile
      - run: pnpm build
      - run: pnpm typecheck
      - run: pnpm test
```

Do not keep sibling checkout complexity if npm dependency works.

## 4.2 CE CI

CE CI should be able to run normally:

```yaml
- run: pnpm install --frozen-lockfile
- run: pnpm contracts:check
- run: pnpm db:validate
- run: pnpm typecheck
- run: pnpm build
- run: pnpm test
```

It should not need sibling checkout of agent/contracts.

# Important Failure Handling

## If npm scope is unavailable

If publishing fails because `@xtrape` scope is not available or permission is missing:

```text
Stop.
Report exact npm error.
Do not rename packages without confirmation.
```

Possible alternatives require owner decision:

```text
1. Create/claim npm org @xtrape.
2. Use @xtrape-com scope.
3. Use unscoped package names temporarily.
```

Do not decide automatically.

## If package version already exists

Do not overwrite.

Increment prerelease:

```text
0.1.0-public-review.1
```

Update agent and CE dependencies accordingly.

## If pnpm install fails in CE

Classify failure:

```text
1. missing npm package
2. version not found
3. lockfile mismatch
4. workspace config mismatch
5. actual type/build failure
```

Fix based on actual error.
