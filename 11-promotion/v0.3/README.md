# xtrape-capsule v0.3 推广发布稿

本压缩包包含 4 个平台的新发布稿：

1. `reddit-v0.3-discussion.md`  
   Reddit 讨论帖，建议发到 r/selfhosted / r/devops / r/docker / r/opensource 等社区。重点是讨论问题，不要硬推项目。

2. `devto-v0.3-article.md`  
   dev.to 英文长文，已包含 frontmatter。建议使用标题：`What Comes After Docker Compose for Small Teams?`

3. `juejin-v0.3-article.md`  
   掘金中文技术文章，偏场景 + 架构解释。建议配合截图和 Quick Start 链接发布。

4. `zhihu-v0.3-article.md`  
   知乎中文文章/回答稿，偏问题讨论和观点表达。也可以改成回答已有问题。

建议发布顺序：

1. Hacker News / GitHub Release 完成后，先发 dev.to。
2. Reddit 发讨论帖，不要只贴链接。
3. 掘金发中文技术文章。
4. 知乎优先作为问题回答发布；如果没有合适问题，再作为文章发布。

核心叙事：

> Docker Compose 负责运行服务，xtrape-capsule 探索 Kubernetes 之前的轻服务治理层。
